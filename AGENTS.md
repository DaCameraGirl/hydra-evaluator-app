# Repo Instructions

This repo is maintained by Angela Hudson / DaCameraGirl.

## Workflow

- Treat requested repo work like real developer work: inspect status, keep changes scoped, run checks, commit intentional changes, push branches, and open PRs when asked to publish or share.
- Prefer branch-based work instead of committing straight to `main`.
- Update README, package metadata, and deployment notes when the app purpose changes.
- Use concise PR notes with what changed, how it was checked, and any follow-up issues.

## Hydra Evaluation Style

- The app supports Project Hydra image-response evaluation work.
- Justifications should use Angela's preferred format: `Response A/B is better than Response A/B.` followed by specific prompt-based reasons.
- Keep wording plain, direct, and non-technical for Hydra rating explanations.
- Do not use em dashes in generated justification text.
- Separate instruction following from correctness when judging images.
- Pay close attention to whether a response preserves the original subject, pose, background, objects, and other prompt invariants.
